# Доменная модель TaskMate

[← Назад: Архитектура](architecture.md) | [Далее: Backend](backend.md)

## Обзор

Доменная модель TaskMate построена вокруг концепции задач в автодилерах. Система поддерживает многоарендность (multi-tenant), где каждый автодилер имеет собственный набор данных и timezone.

## Роли пользователей

Ролевая модель основана на уровнях доступа (число = уровень):

| Роль | Уровень | Описание | Доступ |
|------|---------|----------|--------|
| employee | 1 | Сотрудник | Выполнение назначенных задач |
| observer | 2 | Наблюдатель | Просмотр задач дилера |
| manager | 3 | Менеджер | Создание/управление задачами, верификация |
| owner | 4 | Владелец | Полный доступ ко всем дилерам |

### Права доступа

- **employee**: Только свои задачи и смены
- **observer**: Просмотр всех задач дилера (только чтение)
- **manager**: Управление задачами назначенных сотрудников
- **owner**: Полный доступ ко всем дилерам или конкретному

## Задачи (Tasks)

### Типы задач

| Тип | Описание | Требования |
|-----|----------|------------|
| `notification` | Уведомление | Только просмотр, без действий |
| `completion` | Выполнение | Отметка о выполнении |
| `completion_with_proof` | Выполнение с доказательствами | Загрузка файлов + комментарий |

### Структура задач

- **individual**: Задача для одного сотрудника
- **group**: Групповая задача для нескольких сотрудников

### Статусы задач

| Статус | Описание |
|--------|----------|
| `pending` | Создана, ожидает выполнения |
| `in_progress` | Выполняется сотрудником |
| `pending_review` | Ожидает верификации менеджером |
| `verified` | Одобрена менеджером |
| `rejected` | Отклонена менеджером |
| `archived` | Архивная (выполнена >10 мин назад) |

## Workflow задач

```
Создание → Выполнение → Верификация → Архивация
    ↓         ↓            ↓          ↓
 Manager   Employee    Manager   Scheduler
```

### Детальный процесс

1. **Создание** (Manager/Owner)
   - `POST /tasks` → `TaskService::createTask()`
   - Статус: `pending`
   - Автоматическое назначение через `TaskAssignment`

2. **Выполнение** (Employee)
   - `PATCH /tasks/{id}/status` + файлы доказательств
   - Статус: `pending_review`
   - Для `completion_with_proof`: загрузка до 5 файлов (200MB total)

3. **Верификация** (Manager)
   - `POST /task-responses/{id}/approve` или `reject`
   - При reject: удаление файлов, запись в `VerificationHistory`
   - Статус: `verified` / `rejected`

4. **Архивация** (Scheduler)
   - `tasks:archive-completed` каждые 10 мин
   - Перемещение выполненных задач старше 10 мин в архив

## Многоарендность (Multi-tenant)

### Автодилеры (AutoDealership)

- Каждый дилер — отдельный тенант
- Собственный timezone (например, `+05:00` для Екатеринбурга)
- Изоляция данных через `dealership_id`

### Timezone управление

```php
// Backend: получение timezone с fallback
$timezone = SettingsService::getTimezone($dealershipId);
// 1. AutoDealership.timezone
// 2. settings.global_timezone  
// 3. '+05:00' (дефолт)
```

### Гибридный подход к датам

| Тип | Хранение | Пример |
|-----|----------|--------|
| Моменты времени | UTC | `2026-01-27T10:00:00Z` |
| Календарные дни | Дата + timezone | `2026-01-27` + `+05:00` |

Пример: выходные дни определяются по timezone дилера, а не по UTC.

## Файловое хранилище

### Доказательства задач

- **Хранилище**: Приватный disk `task_proofs`
- **Путь**: `storage/app/private/task_proofs/`
- **Доступ**: Только через signed URLs (TTL 60 мин)
- **Лимиты**: 
  - 5 файлов на задачу
  - 200MB total
  - Изображения: 5MB
  - Видео: 100MB
  - Документы: 50MB

### Валидация файлов

- Magic bytes проверка (не только расширение)
- Типы: JPEG, PNG, MP4, PDF, DOCX, etc.
- Асинхронная обработка через RabbitMQ

## Генераторы задач

### Шаблоны задач

- Автоматическое создание задач по расписанию
- Интеграция с календарем выходных дней
- Учет timezone дилера

### Очереди

| Очередь | Worker | Описание |
|---------|--------|----------|
| `task_generators` | svc-worker-generator | Генерация из шаблонов (каждые 5 мин) |
| `proof_upload` | svc-worker-proof | Загрузка индивидуальных файлов |
| `shared_proof_upload` | svc-worker-shared | Групповые доказательства |
| `file_cleanup` | svc-worker-cleanup | Очистка временных файлов (каждый час) |

## Смены (Shifts)

### Структура смен

- Назначение сотрудников на временные интервалы
- Учет рабочих часов для отчетности
- Интеграция с задачами (фильтрация по смене)

### Связи

- `User` ↔ `Shift` (многие-ко-многим через `UserShift`)
- `Task` может быть привязана к смене

## Ключевые сущности

### User
- `id`, `name`, `email`, `role`
- Связи: `dealership_id`, `shifts`, `assigned_tasks`

### Task
- `id`, `title`, `description`, `type`, `status`
- Связи: `creator_id`, `dealership_id`, `assignments`

### TaskAssignment
- `task_id`, `user_id`, `assigned_at`

### TaskProof
- `task_id`, `file_path`, `file_name`, `mime_type`
- Signed URL для доступа

### AutoDealership
- `id`, `name`, `timezone`, `settings`

### VerificationHistory
- `task_id`, `verifier_id`, `action` (approve/reject), `comment`

## Бизнес-правила

### Валидация

- Задачи могут назначаться только на активных сотрудников
- Файлы доказательств обязательны для `completion_with_proof`
- Верификация только менеджерами с правами на сотрудника

### Аудит

- Все изменения логируются через `Auditable` trait
- История верификаций сохраняется

### Безопасность

- Проверка dealership_id во всех запросах
- Role-based access через Laravel Policies
- SoftDeletes для безопасного удаления
